## Sidebar Tabs

### Chat Tab
Your primary workspace. Communicate with the AI to build your app.

### Settings Tab
Configure your app's identity, permissions, and capabilities. See [Project Settings](/features/project-settings).

### Database Tab
View and manage your Supabase database tables (if connected). See [Supabase](/integrations/supabase/overview).

### Deploy Tab
Publish your app to TestFlight for beta testing. See [Publishing](/publishing/overview).

## Sending Messages

### Basic Input

Type your message and:
- **Enter** â€” Send message
- **Shift+Enter** â€” New line
- **Cmd+Enter** â€” Quick send

### Attaching Images

Drag and drop images into the chat:
- Up to **5 images** per message
- Formats: PNG, JPG, JPEG, GIF, WebP, HEIC

See [Using Images](/features/using-images) for best practices.

---

## Message Best Practices

<Tip>
  **Be specific.** Instead of "make it better," try "increase the button height to 50pt and add a subtle shadow."
</Tip>

<Tip>
  **One thing at a time.** Ask for one feature or change per message for best results.
</Tip>

<Tip>
  **Reference what exists.** Say "change the blue header" instead of "change the header" so the AI knows exactly what you mean.
</Tip>

## Understanding AI Responses

### When Things Work

You'll see:
1. Todo list with items being checked off
2. Tool cards showing files being created/edited
3. App rebuilding in the Simulator
4. Success message when complete

### When Errors Occur

Build failures are handled automatically:
1. AI detects the error
2. Shows what went wrong
3. Proposes a fix
4. Applies fix and rebuilds

Don't panic â€” errors are normal. See [Debugging Guide](/guides/debugging) for details.

### When AI Needs Clarification

The AI may ask questions:
- "Did you mean the header at the top or the section title?"
- "What color for the button?"
- "Should this save locally or to the cloud?"

Answer naturally to continue.

## Conversation Management

### Scrolling History

Your conversation is saved. Scroll up to see:
- Previous requests
- What the AI built
- Earlier iterations

### Clearing the Plan

To start fresh with a new approach:
```
/planclear
```

This clears the current plan but preserves your code.

### Starting Fresh Conversation

If the conversation gets too long:
1. Clear conversation option in menu
2. Confirm
3. Your code is preserved â€” only chat history clears

<Note>
  Clearing conversation doesn't delete your app. Only the chat history is cleared.
</Note>

## Tips for Effective Building

### Start Simple

Begin with core functionality. Add polish later.

### Test Frequently

Check after each change. Catching issues early saves time.

### Use Images

Visual references reduce iterations. See [Using Images](/features/using-images).

### Give Feedback

If something isn't right, tell the AI:
- "That's not what I meant..."
- "Make it more..."
- "Change the color to..."
